## 最近研究成果
1. 似乎最近扒的GTDemo.apk在同CPU（RK3588）环境下可以正常打开（不同CPU则闪退），但是显示不正常（如图）：
   <img width="1302" height="800" alt="image" src="https://github.com/user-attachments/assets/3e6c3d12-75a9-46d0-92ea-36193fcbab81" />
（测试设备为一台触屏跑步机，所以图片右边有它的悬浮控件，忽略即可）
也就是说，或许这个应用和订餐机的定制系统有一定的关系，但是不知道能否解决回跳问题。
3. 订餐机使用第三方应用安装器防霸机似乎不太可行，MT管理器之所以在某些设备上可以直接绕过系统安装器安装其实是因为APPS目录有写权限：
   <img width="1302" height="800" alt="image" src="https://github.com/user-attachments/assets/ca1e9796-ebcf-44ef-bf4b-9c0749939425" />
4. 如果先让订餐机连接WiFi启用ADB，然后通过命令固定ADB端口，使其在WiFi断开回到网线时依旧能保持开启，则有可能通过ADB让Shizuku可用，但是未经测试，如果有机会可以尝试。
5. 关于串口输出，如果有电脑可能可以尝试开一个安卓x86虚拟机并安装houdini兼容层，然后用它运行点餐APP，以获取它的串口输出并研究是否与外控有关，但是现在的问题是IP不对直接闪退，所以这种方法并没有太多可行性。（如果当初的2.0.0还在，或许可以跑起来？）
